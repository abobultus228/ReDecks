# ReDecks

Мобильное приложение для автоматического открытия колод на [Remanga](https://remanga.org) с умным выбором карт по коллекции.

## Возможности

## Колоды (автооткрытие)

Открывает любое количество колод подряд, либо все доступные разом, автоматически определяет лучшую карту по заданной коллекции. При неоднозначной ситуации показывает оверлей с картинками всех выпавших карт, позволяя выбрать самому, учитывает 4-ю карту, если аккаунт премиальный.


## Главы (сброс прочтения)

Находит все прочитанные тобой главы выбранного тайтла (по ссылке или через
поиск) и снимает отметку «прочитано» — главы снова становятся новыми.

## Карты (список персонажей)

Показывает всех персонажей тайтла и количество карт у каждого. Есть фильтр
по количеству карт: переключатель вкл/выкл, поля «от» и «до» и два ползунка.
Тап по персонажу открывает его страницу на remanga.

## Правила выбора для автооткрытия колод

Базовый приоритет фиксирован и не зависит от настроек:

> Если среди выпавших карт есть **ровно одна новая приоритетная** — она выбирается автоматически всегда.

Три настройки управляют поведением в неоднозначных ситуациях.

---

### 1. Приоритетные выпали, но уже в наличии

Срабатывает когда **все** выпавшие приоритетные карты уже есть в коллекции (ни одной новой приоритетной нет).

| Вариант | Поведение |
|---|---|
| **Ручной выбор** | Процесс останавливается, открывается оверлей — выбираешь карту сам |
| **Взять неприоритетную новую, иначе приоритетную** | Берёт случайную новую неприоритетную карту. Если таких нет — берёт приоритетную повторку (с учётом правила 2, если их несколько) |
| **Всё равно взять приоритетную** | Берёт приоритетную карту, даже если она уже есть (с учётом правила 2, если их несколько) |

---

### 2. Несколько приоритетных одного статуса

Срабатывает когда нужно выбрать приоритетную карту, но их **2 или больше с одинаковым статусом**:
- 2+ новых приоритетных одновременно
- 2+ owned приоритетных (при выборе повторки через правило 1)

| Вариант | Поведение |
|---|---|
| **Ручной выбор** | Процесс останавливается, открывается оверлей — выбираешь карту сам |
| **Выбрать случайную приоритетную** | Берёт случайную из подходящих приоритетных |

> **Примеры:**
> - 1 новая прио + 1 старая прио → правило 2 **не** срабатывает, берётся новая автоматически
> - 2 новых прио + 2 неприо → правило 2 **срабатывает**
> - 1 новая прио + 2 старых прио → правило 2 **не** срабатывает, берётся единственная новая

---

### 3. Нет приоритетных среди выпавших

Срабатывает когда в выпавших картах **нет ни одной приоритетной**.

| Вариант | Поведение |
|---|---|
| **Ручной выбор** | Процесс останавливается, открывается оверлей — выбираешь карту сам |
| **Взять новую неприоритетную, иначе рандом** | Берёт случайную новую неприоритетную карту. Если все уже есть — берёт случайную из выпавших |


---

## Навигация

После авторизации внизу появляется панель навигации с тремя разделами:

- **Колоды** — открытие колод
- **Главы** — сброс прочитанных глав тайтла
- **Карты** — список персонажей тайтла и число карт у каждого
